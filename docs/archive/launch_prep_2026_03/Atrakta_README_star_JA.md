# Atrakta

**AIコーディングを安全に、再現可能に、そして観測可能に。**

Atrakta は、\
**Claude Code / Codex / Cursor / Antigravity / VS Code / etc** などの AI コーディング環境を横断して、\
AI開発を安全に運用するための **軽量ランタイムレイヤー** です。

AIによる開発に **状態管理・安全契約・イベントログ** を導入します。

------------------------------------------------------------------------

## ⭐ なぜ Atrakta が必要なのか

AIコーディングツールは非常に強力ですが、実際のプロジェクトでは次のような問題が起こります。

  問題                   起きること
  ---------------------- ----------------------------------
  設定ドリフト           IDE/ CLI / CI で挙動が変わる
  セッションの不安定性   AIの判断が毎回変わる
  デバッグ困難           自動化失敗の原因が追えない
  破壊的変更             AIが危険な変更をする可能性

Atrakta はこれらを解決する **AI開発の運用レイヤー** です。

> **AIコーディングのための Git のような存在**

------------------------------------------------------------------------

## 🚀 Atraktaの特徴

### 再現可能なAIワークフロー

AIの作業状態を保存し、\
**セッションをまたいで再現可能**にします。

------------------------------------------------------------------------

### ツール横断の一貫した挙動

各種 IDE / CLI が\
**同じ運用ルールで動作**します。

------------------------------------------------------------------------

### AI操作の完全ログ

すべてのAIアクションが\
**イベントログとして記録**されます。

------------------------------------------------------------------------

### セーフティコントラクト

AIによる破壊的変更を\
**契約ルールで防止**します。

------------------------------------------------------------------------

### タスクグラフ管理

AIの進捗を\
**構造的に追跡できます。**

------------------------------------------------------------------------

## ⚡ クイックスタート（3分）

### macOS / Linux

``` bash
curl -fsSL https://raw.githubusercontent.com/afwm/Atrakta/main/scripts/install.sh | bash
atrakta init --interfaces cursor
```

------------------------------------------------------------------------

### Windows

リリースページからバイナリをダウンロードして実行

``` powershell
$targetDir = "$env:USERPROFILE\AppData\Local\Programs\atrakta"
New-Item -ItemType Directory -Force $targetDir | Out-Null
Copy-Item .\atrakta.exe "$targetDir\atrakta.exe" -Force
```

初期化

    atrakta init --interfaces cursor

------------------------------------------------------------------------

## 📦 初期化で作成されるファイル

`atrakta init --interfaces <id>` を実行すると、\
AI開発の状態管理レイヤーが作成されます。

    AGENTS.md

    .atrakta/
      contract.json
      events.jsonl
      state.json
      progress.json
      task-graph.json

これらにより AI作業が

-   再開可能
-   追跡可能
-   再現可能
-   安全

になります。

------------------------------------------------------------------------

## 🧭 基本ワークフロー

日常で使うコマンドは3つだけです。

``` bash
atrakta start --interfaces cursor
atrakta resume
atrakta doctor
```

  コマンド   役割
  ---------- ------------------
  start      AIセッション開始
  resume     前回の作業を再開
  doctor     状態診断

------------------------------------------------------------------------

## 🧠 アーキテクチャ

Atrakta は\
AIツールとプロジェクトの間に入ります。

    AIツール（IDE / CLI / CI）
               ↓
            Atrakta
               ↓
       プロジェクト運用状態

       ├ contract
       ├ events
       ├ progress
       └ task graph

この構造により

-   安全なAI実行
-   決定論的ワークフロー
-   復元可能な自動化

が実現されます。

------------------------------------------------------------------------

## 🔍 可観測性

AIのすべての操作は

    .atrakta/events.jsonl

に記録されます。

これにより

-   AI作業のデバッグ
-   変更履歴の監査
-   開発セッションの再現

が可能になります。
