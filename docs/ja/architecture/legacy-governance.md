# レガシー統治と昇格ルール

## レガシー状態

- reference_only
- partially_mapped
- canonicalized

## 昇格条件

すべて必須:

- 所有権が判明している
- 鮮度が許容範囲内
- 正規マッピングが存在する

メタデータ不明のアセットは参照はできるが自動昇格はできない。

## ドリフト条件

- 正規との衝突
- 古いタイムスタンプまたは古いレビュー
- マップ先の欠落
- 重複ガイダンスのリスク
- 非推奨だがまだ参照されている

ドリフトを黙って無視してはならない。
レガシー衝突が常に warn-only で終わってはならない。厳格なエスカレーション経路が必要。

## リゾルバ API

`resolve_legacy_status(asset) -> legacy_status_decision`
