# 監査の整合性と保持ポリシー

## 整合性レベル

- A0: 追記のみ（append-only）
- A1: 追記に加えイベントごとの整合性ハッシュ
- A2: 追記に加え前ハッシュチェーン
- A3: 追記に加えチェーン検証とチェックポイント

整合性レベルはストレージ実装から論理的に独立した契約である。

## 保持と保護

- 追記のみは必須
- コア監査の先頭は GC 対象にしない
- マニフェストと投影のリンクは通常の GC 外とする

## アーカイブルール

- まずドライラン
- 破壊的クリーンアップは提案のみ

監査保証の不足は、厳格トリガーおよび失敗ルーティングにつなげなければならない。

## リゾルバ API

`resolve_audit_requirements(action) -> audit_requirements_decision`
