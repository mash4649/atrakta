# 概念カバレッジマトリクス（リフレッシュトラック）

本マトリクスは、v0 リフレッシュの概念が、過去の Atrakta ラインで扱われてきた
主要な運用意図をどう覆うかを定義する。

## カバレッジのルール

- 目標は概念カバレッジと挙動としての十分性である。
- 本トラックではコマンド／データのパリティは要求しない。

## 意図のマッピング

| 運用意図 | v0 の概念サーフェス | 検証 |
|---|---|---|
| 初回の安全なオンボーディング | `run` のオンボーディング分岐（`detect -> propose -> accept`） | `cmd/atrakta` の run テスト（承認ゲートと永続化チェック） |
| 決定的な日次実行 | `run` の通常分岐と順序付きリゾルバパイプライン | 決定的リプレイテストと run E2E |
| 明示的な書き込み制御 | `--apply` と `--approve` ゲート、管理対象のみの変更 | run の apply テストと変更エンベロープの検証 |
| 監査の追跡可能性 | 追記のみの監査チェーン（`run_execute`） | audit append／verify テスト |
| 失敗からの回復姿勢 | 終了コード契約（`0/1/2/3`）と次アクションのヒント | アダプタループテストと run のスキーマ検証 |

## 受け入れ条件

本スコープの v0 リフレッシュは、次が満たされたとき受け入れられる:

1. 上記の意図がすべてテストで示されること
2. `run` が単一の運用入口のままであること
3. 入力／承認不足のとき、安全性不変条件がフェイルクローズのままであること
