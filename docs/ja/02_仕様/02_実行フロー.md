# 実行フロー

[English](../../en/02_spec/02_execution_flow.md) | [日本語](./02_実行フロー.md)


## `start` の順序（Fast Path First, Strict Path On Demand）

1. contract 読込/初期化と設定正規化
2. Start Snapshot Fast Gate 判定（`runtime/meta.v2.start_fast_v2`）
3. fast hit かつ managed artifact が整合している場合:
   - `VerifyChainCached` 実行
   - `start_fast_hit` / `step(DONE)` / checkpoint 更新
   - strict pipeline を省略して終了
4. fast miss または drift 検知時は strict path へ昇格:
   - prompt policy 既定ファイル初期化（必要時）
   - events chain 検証（`schema_version = 2` 固定）
   - git setup / pre snapshot
5. `AGENTS.md` 自動生成（必要時）と context resolver（`nearest_with_import`）
6. repo map 読込/再生成（予算・更新間隔制御）と `repo_map` イベント記録
7. routing 決定（`task_category -> worker/quality`）、`progress.json` 初期化（必要時）
8. interface 解決（明示 -> trigger -> auto-last -> detect観測 -> 対話入力）
9. detect 実行
10. projection 生成 + subworker phaseA + single_writer_queue 統合
11. plan 構築（required_permission算出 + task DAG生成/保存）
12. prompt policy 条件適用（Goal prefix）
13. 承認が必要なら approval
14. security preflight（`read_only` なら mutation をBLOCK）
15. apply 実行（task graph トポロジカル順）
16. gate（safety + quick/heavy, route quality連携）
17. state/progress 更新、git post snapshot/checkpoint 記録、step event 記録
18. `start_fast_v2` スナップショット更新（次回 fast gate 用）
19. `DONE` 後に閾値型 auto GC をベストエフォートで非同期起動（クリティカルパス外）

## strict path へ昇格する代表条件

- fast snapshot 不一致（contract hash / workspace stamp / interface set / feature / config key）
- strict interval 超過（既定10分）
- managed artifact 欠損/ドリフト
- fast gate 利用不可（meta 破損など）

## ブロック条件の代表例

- active feature と指定 feature の不一致
- interface 未解決（非対話で `NEEDS_INPUT`）
- context import 循環 / 深度超過
- prompt policy 必須時の欠落・不正
- security profile と required_permission の不整合
- apply 失敗
- gate 失敗
- events/state 破損で復旧不能
