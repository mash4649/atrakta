## Antigravity workflow/rules

- Antigravity は現状 `.antigravity/AGENTS.md` の native projection を提供します。
- workflow/rules 系の拡張は extension projection として出力します。
  - `.extensions/workflows/*.md`
  - `.extensions/hooks/workflow.before_start.md`
  - `.extensions/hooks/workflow.after_apply.md`
- unsupported な native workflow/rule への in-place 注入は行いません。

## Extension projection policy

- `extensions.mcp` -> `.extensions/mcp/*.md`
- `extensions.plugins` -> `.extensions/plugins/*.md`
- `extensions.skills` -> `.extensions/skills/*.md`
- `extensions.workflows` -> `.extensions/workflows/*.md`
- `extensions.hooks` -> `.extensions/hooks/*.md`

interface-native の拡張 projection がない場合は、deterministic な fallback/link markdown を生成し、`.atrakta/extensions/manifest.json` に記録します。

## Unsupported policy

- silent ignore しない
- field path と理由を warning で可視化する（例: `unsupported_extension_projection`）
- strict runtime check では drift/integration warning が blocking に昇格しうる
- extension object（`mcp/plugins/skills/workflows/hooks`）の reverse sync は proposal-only

## Optional templates

- `contract-json`: `<projectionDir>/CONTRACT.json`
- `atrakta-link`: `<projectionDir>/.atrakta-link`
- `cursor-rule`: `<projectionDir>/rules/00-atrakta.mdc`（Cursor）

`max_per_interface`（既定3）を超える optional template 指定は拒否されます。
