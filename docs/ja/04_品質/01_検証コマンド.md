# 検証コマンド

[English](../../en/04_quality/01_verification_commands.md) | [日本語](./01_検証コマンド.md)


## 基本検証

```bash
./scripts/dev/verify_loop.sh
```

`go test ./...` の失敗時に `doctor` を挟んで1回だけ再試行する、最小の復旧付き検証ループです。

## Provisional検証

```bash
./scripts/dev/verify_provisional.sh
```

実施内容:

- `gofmt` + 全テスト
- wrapper fast-path ベンチ閾値判定（<50ms）
- Linux/Windows クロス `go test -run '^$'`
- Linux/Windows バイナリ生成確認

## Windows native parity gate

```bash
./scripts/dev/verify_windows_native_parity.sh
```

Windows ネイティブ runner 上で strict parity フロー（`TestWindowsNativeParity`）を実行し、CI/release で必須ゲートとして扱います。

skip 条件は限定します:

- 既定: Windows native 以外では失敗
- 明示 override のみ許可: `ATRAKTA_ALLOW_WINDOWS_PARITY_SKIP=1`

## Phase2検証

```bash
./scripts/dev/verify_phase2.sh
```

追加内容:

- detect/apply スケーリングベンチ
- p95/p99 パフォーマンス回帰ゲート（fail-closed）
- projection 線形スケール検証
- H7 migration replay determinism テスト
- events schema v2 整合テスト（`migrate check`）
- LinuxバイナリのDocker実行確認

## パフォーマンス回帰ゲート

```bash
./scripts/dev/verify_perf_gate.sh
```

内容:

- `BenchmarkApplyScaling/ops_300` の p95/p99 しきい値判定
- `BenchmarkBuildNoopManagedScaling/managed_1000` の p95/p99 しきい値判定
- `BenchmarkWrapperFastPath` の p95/p99 しきい値判定
- `BenchmarkStartSteadyState` の p95/p99 しきい値判定
- `TestSLORepoMapTokenBudgetRespected`（トークン予算）
- `TestSLOWrapperFastPathHitRate`（fast-path命中率）
- 閾値超過時は即失敗（fail-closed）

CI運用:

- GitHub Actions（`.github/workflows/ci.yml`）で `verify_loop` + `verify_perf_gate` を `push/pull_request` ごとに実行

## 障害注入テスト

- `go test ./internal/events -run '^TestFaultInjection'`
- `go test ./internal/util -run '^TestFaultInjectionWithFileLock'`
- `verify_loop` に含まれるため、通常CIで常時実行される

## ソーク（長時間連続運用）

```bash
# 任意分数での短時間実測
./scripts/dev/soak.sh --minutes 10

# 24h / 72h プロファイル
./scripts/dev/soak_24h.sh
./scripts/dev/soak_72h.sh
```

注意:

- GitHub Hosted Runner の実行時間上限で 24h/72h は完走できないため、自己管理環境での実行を前提とする。
