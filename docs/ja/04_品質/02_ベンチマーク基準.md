## Fast Path

- 指標: `BenchmarkWrapperFastPath`
- 合格条件: `ns/op < 50,000,000`（50ms未満）
- 計測方法: `-count=5` の最終値を自動抽出して判定

## SLOゲート（fail-closed）

- `BenchmarkApplyScaling/ops_300`: p95/p99
- `BenchmarkBuildNoopManagedScaling/managed_1000`: p95/p99
- `BenchmarkWrapperFastPath`: p95/p99
- `BenchmarkStartSteadyState`: p95/p99
- `TestSLORepoMapTokenBudgetRespected`: `repo_map.used_tokens <= budget`
- `TestSLOWrapperFastPathHitRate`: `hit_rate >= 0.95`

注記:

- `BenchmarkStartSteadyState` は snapshot fast gate が有効な定常系を測る。
- strict interval 経過時は strict path に昇格するため、定常値より遅いスパイクは仕様内。

## 運用ルール

- 閾値違反時はスクリプトを即失敗にする
- p95/p99 + token/hit-rate を `verify_perf_gate.sh` で fail-closed 判定する
- 速度最適化時も決定論と安全ゲートは維持する
- DAG task graph を導入しても、依存なしケースの fast-path を維持する
- edit safety（Go AST / JSON parse）導入後も apply スケール劣化が許容内であることを確認する
- `repair` は通常 `start` の自動クリティカルパスに入れない（明示コマンドで実行する）
