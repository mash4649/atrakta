## 1. バイナリ配置

`atrakta` をPATH上に配置します。

配布バイナリ利用が前提です。  
推奨配置先:

- macOS / Linux: `~/.local/bin/atrakta`
- Windows: `%USERPROFILE%\\AppData\\Local\\Programs\\atrakta\\atrakta.exe`

推奨の1コマンド導入（macOS / Linux）:

```bash
curl -fsSL https://raw.githubusercontent.com/mash4649/atrakta/main/scripts/install.sh | bash
```

`install.sh` は OS/arch 自動判別、対応リリース資産の取得、チェックサム検証、`~/.local/bin` への配置、macOS の quarantine 属性解除を自動で行います。

Releases から取得後の配置例:
アーカイブ展開後の実行ファイル名は `atrakta`（Windows は `atrakta.exe`）で統一されます。

```bash
# macOS / Linux
mkdir -p ~/.local/bin
install -m 0755 ./atrakta ~/.local/bin/atrakta
hash -r
atrakta --help
```

```powershell
# Windows (PowerShell)
$targetDir = "$env:USERPROFILE\AppData\Local\Programs\atrakta"
New-Item -ItemType Directory -Force $targetDir | Out-Null
Copy-Item .\atrakta.exe "$targetDir\atrakta.exe" -Force
$userPath = [Environment]::GetEnvironmentVariable("Path","User")
if ($userPath -notlike "*$targetDir*") {
  [Environment]::SetEnvironmentVariable("Path", "$targetDir;$userPath", "User")
}
where atrakta
```

任意で、利用対象プロジェクト内のソースからビルドして配置する場合:

```bash
# 例: 利用対象プロジェクトが <project> で、ソースが <project>/atrakta にある場合
cd <project>/atrakta
go build -o ~/.local/bin/atrakta ./cmd/atrakta
hash -r
atrakta --help
```

`command not found` になる場合:

```bash
echo "$PATH" | tr ':' '\n' | grep "$HOME/.local/bin"
ls -l ~/.local/bin/atrakta
```

配布バイナリ利用時は Go のインストールは不要です。

## 2. 初回セットアップ

プロジェクトルートで実行:

```bash
# 統一コマンド（対話/非対話とも同じ）
atrakta init --interfaces cursor
```

必要に応じて自動生成:

- `AGENTS.md`
- `.atrakta/contract.json`
- `.atrakta/state.json`
- `.atrakta/events.jsonl`
- `.atrakta/progress.json`
- `.atrakta/task-graph.json`（plan実行時）
- `.atrakta/policies/prompt-min.json`（既定参照時）

## 3. 推奨オプション

- 2の1コマンド実行後は、通常はwrapper/hook/IDE自動起動タスク経由で `start` が自動解決される
- `--interfaces` を省略した場合は、対話入力または `needs_input` で解決する（暗黙デフォルトなし）

段階的導入にしたい場合:

```bash
atrakta wrap install
atrakta hook install
atrakta ide-autostart install
atrakta start --interfaces cursor
```
