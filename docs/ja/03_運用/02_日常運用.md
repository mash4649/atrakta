# 日常運用

[English](../../en/03_運用/02_日常運用.md) | [日本語](./02_日常運用.md)


## 基本運用

- 通常は wrapper 経由で各インターフェースを起動
- VSCode互換IDEでは `ide-autostart` により workspace オープン時に `start` が自動実行される
- 手動 `atrakta start` でも、前回成功ターゲット/トリガー情報から自動解決される
- 大規模リポジトリでは `--map-tokens` / `--map-refresh` で repo map 予算を調整できる
- 大きな変更後に `atrakta doctor` を実行
- 容量が増えた場合は `atrakta gc --scope tmp` で dry-run、問題なければ `--apply`
- 旧イベントログ（schema v1）は非対応。必要なら退避して新規ログで再開する

## feature運用

- 明示featureで実行:

```bash
atrakta start --feature-id feat-auth
```

- active feature が残っている場合、別featureへの切替はブロックされる
- 完了時は `completed_features` に記録される

## 同期運用

- 提案確認:

```bash
atrakta doctor --sync-proposal
```

- 承認適用:

```bash
atrakta doctor --sync-proposal --apply-sync
```
