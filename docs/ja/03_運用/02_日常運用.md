# 日常運用

[English](../../en/03_operations/02_daily_operations.md) | [日本語](./02_日常運用.md)


## 基本運用

- 通常は wrapper 経由で各インターフェースを起動
- VSCode互換IDEでは `ide-autostart` により workspace オープン時に `start` が自動実行される
- 手動 `atrakta start` でも、前回成功ターゲット/トリガー情報から自動解決される
- 大規模リポジトリでは `--map-tokens` / `--map-refresh` で repo map 予算を調整できる
- 大きな変更後に `atrakta doctor` を実行
- 大きな変更後は parity/integration も確認する
- 容量が増えた場合は `atrakta gc --scope tmp` で dry-run、問題なければ `--apply`
- 旧イベントログ（schema v1）は非対応。必要なら退避して新規ログで再開する

### 変更後チェック（推奨）

```bash
atrakta doctor
atrakta doctor --parity
atrakta doctor --integration
atrakta projection status --json
```

## ツール切替時の projection 運用

- インターフェースを切り替えるときは、明示 render を先に実行して drift を減らす

```bash
atrakta projection render --interface cursor
atrakta projection render --interface claude_code
atrakta projection render --interface codex_cli
atrakta projection status --json
```

## feature運用

- 明示featureで実行:

```bash
atrakta start --feature-id feat-auth
```

- active feature が残っている場合、別featureへの切替はブロックされる
- 完了時は `completed_features` に記録される

## 同期運用

- 提案確認:

```bash
atrakta doctor --sync-proposal
```

- 提案を履歴保存（監査/レビュー向け）:

```bash
mkdir -p .atrakta/proposals
atrakta doctor --sync-proposal > .atrakta/proposals/sync-$(date +%Y%m%d-%H%M%S).json
```

- 承認適用:

```bash
atrakta doctor --sync-proposal --apply-sync
```
