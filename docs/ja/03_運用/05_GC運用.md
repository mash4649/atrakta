# GC運用

[English](../../en/03_運用/05_GC運用.md) | [日本語](./05_GC運用.md)


## 方針

- 文脈本体（`state.json`, `progress.json`, checkpoint, auto-state）は削除しない
- `.tmp` は閾値超過時のみ自動GC対象
- `events.jsonl` は proposal-only（自動変更しない）
- すべてのGC実行で dry-run/適用結果をログ化する

## 自動GC（startの外側）

- `start` が `DONE` のときのみ、自動GCをベストエフォートで非同期起動
- 最小間隔（既定60分）未満では自動起動しない
- 閾値未満なら `.tmp` は削除しない

## 手動GC

```bash
# dry-run
atrakta gc --scope tmp

# apply
atrakta gc --scope tmp --apply

# events提案確認（proposal-only）
atrakta gc --scope events
```

## 主な環境変数

- `ATRAKTA_GC_DISABLE=1` 自動GCを停止
- `ATRAKTA_GC_TMP_MAX_BYTES` `.tmp` 閾値（既定 2GiB）
- `ATRAKTA_GC_TMP_RETENTION_DAYS` 優先削除日数（既定 7日）
- `ATRAKTA_GC_AUTO_MIN_INTERVAL_MIN` 自動GC最小間隔（既定 60分）

## ログ

- `.atrakta/runtime/gc-log.jsonl`
  - `tmp.dry_run_delete*`
  - `tmp.applied_delete*`
  - `events.proposals[]`
