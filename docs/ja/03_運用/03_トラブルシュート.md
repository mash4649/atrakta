# トラブルシュート

[English](../../en/03_運用/03_トラブルシュート.md) | [日本語](./03_トラブルシュート.md)


## `events.jsonl corrupted`

- hash chain 不整合が原因
- 旧イベントログ（schema v1）は非対応
- `.atrakta/events.jsonl` を退避し、新規ログで `atrakta doctor` を実行

## `migrate check failed: schema_version unsupported`

- 旧schemaのイベントが残っている
- 旧schemaは最新仕様では非対応
- 旧ログを退避してから再実行

## `task graph invalid`

- `.atrakta/task-graph.json` が壊れているか、手編集で不整合
- `atrakta start` を再実行して task graph を再生成

## `feature switch blocked`

- `progress.json.active_feature` と指定featureが不一致
- 現在featureを完了させるか、復旧手順で整合を取り直す

## 中断後に再開したい

- `atrakta resume` で最新 checkpoint の条件を再利用して再開できる
- 上書きしたい場合は `--feature-id` / `--interfaces` / `--sync-level` を指定

## wrapperが効かない

- `ATRAKTA_WRAP_DISABLE=1` が設定されていないか確認
- `~/.local/bin` のwrapper配置を確認
- `atrakta wrap install` を再実行（必要なら rc にPATH優先設定を自動追記）
- シェル再起動後に `command -v cursor` などで `~/.local/bin` 側が先に解決されるか確認

## hookが暴走する

- 一時停止: `ATRAKTA_HOOK_DISABLE=1`
- 削除: `atrakta hook uninstall`

## `command not found: atrakta`

- バイナリ未配置か PATH 未設定が原因
- macOS / Linux は1コマンド導入で復旧可能:
  - `curl -fsSL https://raw.githubusercontent.com/mash4649/atrakta/main/scripts/install.sh | bash`
- macOS / Linux:
  - `mkdir -p ~/.local/bin`
  - `install -m 0755 ./atrakta ~/.local/bin/atrakta`
  - `hash -r`
  - `command -v atrakta`
- Windows (PowerShell):
  - `$targetDir = "$env:USERPROFILE\AppData\Local\Programs\atrakta"`
  - `Copy-Item .\atrakta.exe "$targetDir\atrakta.exe" -Force`
  - ユーザーPATHに `$targetDir` を追加
  - `where atrakta`

## strict時に期待挙動と違う

- `sync-level=2` では AGENTS由来提案は意思決定に使われない

## `.tmp` が肥大化する

- dry-run確認: `atrakta gc --scope tmp`
- 問題なければ適用: `atrakta gc --scope tmp --apply`
- 自動GC停止中なら `ATRAKTA_GC_DISABLE` 設定を確認

## `events.jsonl` が大きい

- `atrakta gc --scope events` で提案を確認
- 現行では proposal-only（自動変更しない）
