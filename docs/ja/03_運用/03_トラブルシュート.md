# トラブルシュート

[English](../../en/03_operations/03_troubleshooting.md) | [日本語](./03_トラブルシュート.md)


## `events.jsonl corrupted`

- hash chain 不整合が原因
- 旧イベントログ（schema v1）は非対応
- `.atrakta/events.jsonl` を退避し、新規ログで `atrakta doctor` を実行

## `migrate check failed: schema_version unsupported`

- 旧schemaのイベントが残っている
- 旧schemaは最新仕様では非対応
- 旧ログを退避してから再実行

## `task graph invalid`

- `.atrakta/task-graph.json` が壊れているか、手編集で不整合
- `atrakta start` を再実行して task graph を再生成

## `feature switch blocked`

- `progress.json.active_feature` と指定featureが不一致
- 現在featureを完了させるか、復旧手順で整合を取り直す

## 中断後に再開したい

- `atrakta resume` で最新 checkpoint の条件を再利用して再開できる
- 上書きしたい場合は `--feature-id` / `--interfaces` / `--sync-level` を指定

## wrapperが効かない

- `ATRAKTA_WRAP_DISABLE=1` が設定されていないか確認
- `~/.local/bin` のwrapper配置を確認
- `atrakta wrap install` を再実行（必要なら rc にPATH優先設定を自動追記）
- シェル再起動後に `command -v cursor` などで `~/.local/bin` 側が先に解決されるか確認

## hookが暴走する

- 一時停止: `ATRAKTA_HOOK_DISABLE=1`
- 削除: `atrakta hook uninstall`

## `command not found: atrakta`

- バイナリ未配置か PATH 未設定が原因
- macOS / Linux は1コマンド導入で復旧可能:
  - `curl -fsSL https://raw.githubusercontent.com/mash4649/atrakta/main/scripts/build/install.sh | bash`
- macOS / Linux:
  - `mkdir -p ~/.local/bin`
  - `install -m 0755 ./atrakta ~/.local/bin/atrakta`
  - `hash -r`
  - `command -v atrakta`
- Windows (PowerShell):
  - `$targetDir = "$env:USERPROFILE\AppData\Local\Programs\atrakta"`
  - `Copy-Item .\atrakta.exe "$targetDir\atrakta.exe" -Force`
  - ユーザーPATHに `$targetDir` を追加
  - `where atrakta`

## strict時に期待挙動と違う

- `sync-level=2` では AGENTS由来提案は意思決定に使われない

## `projection drift detected` / parity drift

- 原因: 管理対象 projection が欠損・改変・hash 不一致
- 確認:
  - `atrakta doctor --parity`
  - `atrakta projection status --json`
- repair:
  - `atrakta projection repair --all`
- fallback:
  - 破損 interface だけ再生成: `atrakta projection render --interface <id>`

## `projection manifest is missing`

- 原因: `.atrakta/projections/manifest.json` が未生成/削除
- 確認:
  - `atrakta doctor --parity`
- repair:
  - `atrakta projection render --all`
- fallback:
  - `atrakta start --interfaces <id>` で再生成

## `managed_block_corruption`

- 原因: managed block を手編集した、または include/append 範囲が崩れた
- 確認:
  - `atrakta doctor --parity`
  - `atrakta doctor --integration`
- repair:
  - `atrakta projection repair --all`
- fallback:
  - 当該ファイルの user-managed 部分を残して managed 範囲を再生成

## `parity blocked`

- 原因: contract/parity surface/manifest 整合が fail-closed 条件に抵触
- 確認:
  - `atrakta doctor --parity --json`
- repair:
  - 出力された blocking issue の `repair_hint` に従う
- fallback:
  - strict 実行前に `atrakta projection status --json` で差分確認

## `integration blocked`

- 原因: overwrite risk / append failure / include missing などの brownfield 衝突
- 確認:
  - `atrakta doctor --integration --json`
- repair:
  - `atrakta projection repair --all` または `atrakta init --mode brownfield --no-overwrite`
- fallback:
  - `.atrakta/proposals/*.patch` をレビューして手動適用

## `unsupported extension projection`

- 原因: extension は有効だが native projection が未対応
- 確認:
  - `atrakta doctor --integration`
- repair:
  - fallback/link markdown（`.extensions/*`）を利用する
- fallback:
  - extension を一時的に無効化して次リリースで再有効化

## `.tmp` が肥大化する

- dry-run確認: `atrakta gc --scope tmp`
- 問題なければ適用: `atrakta gc --scope tmp --apply`
- 自動GC停止中なら `ATRAKTA_GC_DISABLE` 設定を確認

## `events.jsonl` が大きい

- `atrakta gc --scope events` で提案を確認
- 現行では proposal-only（自動変更しない）
