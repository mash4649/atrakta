# リリースチェックリスト（Parity/Extensions/Brownfield）

[English](../../en/03_operations/09_release_checklist.md) | [日本語](./09_リリースチェックリスト.md)

このチェックリストを release gate として使います。未達項目が1つでもあればリリースしません。

## 必須ゲート項目

1. [ ] `docs/en/02_spec/06_parity_contract.md` と `docs/ja/02_仕様/06_Parity Contract.md` が存在する。
2. [ ] `docs/en/02_spec/07_extension_surface.md` と `docs/ja/02_仕様/07_Extension Surface.md` が存在する。
3. [ ] `.atrakta/contract.json` に `parity` / `extensions` があり、`v=1` を満たす。
4. [ ] `.atrakta/projections/manifest.json` が projection/start 後に生成される。
5. [ ] `.atrakta/extensions/manifest.json` が projection/start 後に生成される。
6. [ ] 主要3系統の native projection が生成される。
   - Cursor（`.cursor/AGENTS.md`, `.cursor/rules/00-atrakta.mdc`）
   - Claude Code（`CLAUDE.md`, `.claude/*`）
   - Codex（`AGENTS.md`, `.codex/config.toml`）
7. [ ] `atrakta doctor --parity` が drift を正しく検出できる（blocking/warning 区別）。
8. [ ] `atrakta doctor --integration` が brownfield 衝突を正しく検出できる。
9. [ ] AGENTS append/include が idempotent（managed block 増殖なし）。
10. [ ] 次の verify スクリプトが通過する。
    - `./scripts/dev/verify_parity.sh`
    - `./scripts/dev/verify_extensions.sh`
    - `./scripts/dev/verify_brownfield.sh`
    - `./scripts/dev/verify_projection_repair.sh`
11. [ ] fast path / SLO / fail-closed の前提を維持している。
12. [ ] Windows native parity ゲートが成功している。
    - `./scripts/dev/verify_windows_native_parity.sh`

## 推奨実行順

```bash
./scripts/dev/verify_parity.sh
./scripts/dev/verify_extensions.sh
./scripts/dev/verify_brownfield.sh
./scripts/dev/verify_projection_repair.sh
./scripts/dev/verify_perf_gate.sh
./scripts/dev/verify_windows_native_parity.sh
atrakta doctor --parity --json
atrakta doctor --integration --json
atrakta projection status --json
```

## 添付エビデンス

- `scripts/dev/verify_*.sh` の JSON 出力
- `doctor --parity --json` / `doctor --integration --json` のログ
- projection / extension manifest のスナップショット
- Linux 検証と Windows native parity ジョブの CI リンク

## fail-closed ルール

項目未達またはエビデンス不足がある場合はリリースしない。
