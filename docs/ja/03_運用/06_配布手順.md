# 配布手順

[English](../../en/03_operations/06_distribution_guide.md) | [日本語](./06_配布手順.md)


## 方針

- エンドユーザーには事前ビルド済みバイナリを配布する（Go不要）。
- ソースからのビルドは開発者・メンテナ向けのオプションとする。
- リポジトリ本体に配布バイナリをコミットしない。
- `main` への push でリリースを自動生成する（`.github/workflows/release.yml`）。
- リリースタグは CalVer（`YYYY.MM.DD-N`）、プロダクト版は `VERSION`（SemVer）を維持する。

## 自動リリース

- トリガー:
  - `push` to `main`
  - `workflow_dispatch`（手動実行）
- 処理:
  1. 直近リリースから次の CalVer タグを決定
  2. `scripts/build_release_artifacts.sh` で配布物生成
  3. `.github/release.yml` テンプレートで差分リリースノートを生成
  4. GitHub Releases を作成し、パッケージとチェックサムを添付
  5. リリースノートに projection compatibility / parity schema version を含める

## アーティファクト生成

```bash
./scripts/build/build_release_artifacts.sh
```

生成先:

- `.tmp/release/v<version>/packages`
- `.tmp/release/v<version>/checksums.txt`

## 公開手順

1. 通常は自動リリース結果を確認する（手動作成は不要）
2. 失敗時のみ `workflow_dispatch` か手動アーティファクト生成で再実行
3. 利用者向けに更新手順（バイナリ置換 -> parity/integration check）を案内

## 利用者案内テンプレート

1. macOS / Linux 利用者は1コマンド導入を利用可能:
   - `curl -fsSL https://raw.githubusercontent.com/mash4649/atrakta/main/scripts/build/install.sh | bash`
2. Releases から自OS/archのアーカイブを取得（手動導入する場合）
   - 展開後の実行ファイル名は `atrakta`（Windows は `atrakta.exe`）
3. `atrakta`（Windowsは `atrakta.exe`）をPATH配下へ配置
   - macOS / Linux:
     - `mkdir -p ~/.local/bin`
     - `install -m 0755 ./atrakta ~/.local/bin/atrakta`
     - `hash -r`
   - Windows (PowerShell):
     - `$targetDir = "$env:USERPROFILE\AppData\Local\Programs\atrakta"`
     - `New-Item -ItemType Directory -Force $targetDir | Out-Null`
     - `Copy-Item .\atrakta.exe "$targetDir\atrakta.exe" -Force`
     - ユーザーPATHに `$targetDir` を追加
4. `atrakta --help` で動作確認
5. 既存リポジトリ導入では brownfield init を案内
   - `atrakta init --mode brownfield --interfaces cursor --merge-strategy append --agents-mode append --no-overwrite`
6. 導入後確認として以下を案内
   - `atrakta doctor --parity`
   - `atrakta doctor --integration`
   - `atrakta projection status --json`
